# 一、Pod状态排查（必背）
## 1.**Pending**
- 没调度成功
- 资源不足，节点无资源、污点、亲和度不满足
- 排查：`kubectl describe pod`
## 2.**ContainerCreating**
- 正在创建容器
- 镜像拉取慢、存储挂载失败、CNI网络问题
## 3.**ImagePullBackOff / ErrImagePull**
- 镜像拉取失败
- 镜像名错误、私有镜像无认证、仓库不通
## 4.**CrashLoopBackOff**
- 容器启动后奔溃，反复重启
- 程序错误、配置错误、端口占用
- 排查：`kubectl logs`
## 5.**OOMKilled**
- 内存超出限制，被内核杀死
- 退出码137
- 排查：`kubectl describe pod`看Reason
## 6.**Completed**
- 正常退出（一次性任务Job）
- 不是故障
# 二、网络不通排查（高频）
## 1.**Pod不通**
- 节点网络异常
- CNI插件异常（Flannel/Clico）
- NetworkPolicy禁止访问
## 2.**Service不通**
- label/selector 不匹配
- port/targetPort写错
- kube-proxy异常
## 3.**DNS解析失败**
- cordons Pod异常
- 网络策略拦截
- 集群网络问题
# 三、节点异常排查
## 1.**Node NotReady**
- kubelet未运行
- 磁盘/CPU/内存资源满
- 网络不通
## 2.**节点压力**
- 磁盘满
- 内存耗尽
- PID满
# 四、排查四件套命令（背会就够）
## 1.看Pod状态
`kubectl get pod`
## 2.看事件/错误原因
`kubectl describe pod <名称>`
## 3.看程序日志
`kubectl logs <名称>`
## 4.进入容器排查
`kubectl exec -it <名称> -- bash`
## 5.如何查看 K8s 集群、节点、Pod 的资源使用情况？
- 查看**节点资源**：`kubectl top nodes`
- 查看 **Pod 资源**：`kubectl top pods`
- 查看**节点详细资源、分配情况**：`kubectl describe node <节点名>`
- 生产环境用：**Prometheus + Grafana** 看历史监控
# 五、面试一句话口诀
- **Pending = 调度失败**
- **起不来 = 镜像 / 挂载 / 配置**
- **重启 = 程序崩了**
- **OOM = 内存超了**
- **Service 不通 = 标签或端口**
- **DNS 失败 = coredns 或网络策略**
