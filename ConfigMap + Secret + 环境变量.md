# 一、ConfigMap
## 是什么？
**存放普通配置文件、环境变量、参数的地方。**
- 不加密
- 存放**非敏感**配置
## 作用
- 把配置和镜像解藕
- 同一个镜像，不同环境用不同配置
## 使用方式
1. 当作环境变量注入
2. 挂载成文件
# 二、Secret
## 是什么？
**存放敏感信息，会做Base64编码。**
- 密码
- Token
- 密钥 / 证书
## 作用
- 安全存放敏感数据
- 避免写在镜像或代码里
## 使用方式
和ConfigMap一样：
- 环境变量
- 挂载文件
# 三、ConfigMap&Secret（必背）

| 对比   | ConfigMap | Secret          |
| ---- | --------- | --------------- |
| 存放内容 | 普通配置      | 密码、密钥           |
| 是否加密 | 不加密       | Base64编码（不算强加密） |
| 用途   | 配置        | 敏感信息            |
# 四、环境变量/命令行参数
## 环境变量
- 在容器里直接用`$变量`
- 配置简单，适合小参数
## 命令行参数
- 容器启动时传给程序
- Dockerfile里`CMD/CMD/ENTERYPOINT`
# 五、Projected volume
**把四种资源映射到同一目录：**
1.ConfigMap
2.Secret
3.DownwardAPI
4.ServiceAccoutToken
# 六、面试高频（12题）
**1.ConfigMAp作用？**
存储**非敏感配置**，实现配置与镜像解偶。
**2.Secret作用？**
存储**敏感信息**（密码、token、证书）。
**3.Secret安全吗？**
Base64编码，不是加密，只是编码。
真正安全要开**加密Etcd**或外部密钥管理。
**4.ConfigMap/Secret怎么用？**
- 环境变量
- 挂载为文件
**5.ConfigMap更新后，容器会自动更新吗？**
挂载为文件：**会自动更新。**
环境变量：**不会**，必须重启pod。
**6.DownwardAPI干什么？**
把**pod自身信息**（IP、名称、标签、注解）暴露给容器。
**7.ServiceAccounToken干什么？**
给容器里的程序**访问 K8s API**权限。
**8.环境变量和挂载配置文件的区别？**
- 环境变量：简单参数，**不热更新**
- 挂载文件：配置文件，**支持热更新**
**9.热更新配置怎么做？**
ConfigMap/Secret以**文件形式挂载**，应用支持热加载。
**10.Secret和证书怎么用？**
创建 Secret 存放证书 → 挂载到容器 → 程序使用。
**11.为什么不把配置写到镜像里面？**
- 不方便修改
- 不同环境要打不镜像
- 违反**镜像不可变原则**
12.Project Volume类型
- ConfigMap
- Secret
- DownwardAPI
- ServiceAccounToken


# 超级一句话总结

**ConfigMap 存配置，Secret 存密码，

**环境变量简单，挂载文件可热更，**

**Projected 把四种卷合在一起。**
