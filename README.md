# k8s
关于k8s的一些基本知识

# 介绍
一个开源的 容器编排平台，主要用来管理和自动化 容器化应用的部署、扩展和运维。通俗来说，它解决了在大规模环境下运行容器应用时的复杂性问题。
## 核心用途
- 1.容器编排：当你有很多 Docker 容器需要运行时，K8s 能帮你安排这些容器在哪些服务器（节点）上运行，保证资源利用最大化。
- 2.自动化部署：你可以定义应用的“期望状态”，比如“这个服务要运行 3 个副本”，K8s 会自动保证这个状态，即使某个容器挂掉，它也会自动重启或调度到其他节点。
- 3.弹性伸缩：根据流量或资源使用情况，K8s 可以自动增加或减少容器副本数量。
- 4.服务发现与负载均衡：自动分配流量到不同容器实例，不需要手动配置负载均衡器。
- 5.滚动更新与回滚：可以平滑升级应用，而不影响正在运行的服务；如果升级出问题，还能快速回滚。
- 6.资源管理与监控：可以对 CPU、内存等资源进行限制和分配，保证应用稳定运行。
## 使用场景
- 1.微服务架构：管理大量微服务容器。
- 2.高可用应用：需要自动恢复故障容器。
- 3.弹性伸缩应用：按需自动增减实例。
- 4.DevOps & CI/CD：自动化部署、更新和回滚。
## 核心部件
- 1.Pod：最小部署单元，里面可以有一个或多个紧密耦合的容器。
- 2.Node（节点）：运行容器的服务器。
- 3.Deployment：管理 Pod 副本和升级策略。
- 4.Service：提供访问 Pod 的统一入口。
- 5.Namespace：逻辑隔离，用于不同团队或环境。


# K8s两大组件
Master（大脑）+Node（干活）
## 1）Master组件（控制平面）
### 1.etcd
- 分布式键值存储
- **存集群的所有信息：Pod、service、配置、状态......**
- 相当于K8s的数据库
### 2.Kube-apiserver
- 集群**唯一入口**
- 所有操作都必须走它
- 负责认证、鉴权、校正、操作etcd
### 3.Kube-controller-manager
- 各种控制的大总管
- 保证**期望状态=实际状态**
- 比如：副本少了就拉新、节点挂了就迁移Pod
### 4.Kube-schedluer
- 调度器
- 决定：**这个新Pod应该去哪个Node**
- 看资源、标签、污点、亲和度等
## 2）Node组件（工作节点）
### 1.kubelet
- 节点上的**agent**
- 管本机Pod：创建、启动、监控、销毁
- 跟APIserver汇报
### 2.kube-proxy
- 维护节点上的网络规则
- 实现**Service的负载均衡、访问**
- 模式：iptables/IPVS
### 3.容器运行时（CRI）
- 真正跑容器的：contained、Docker等
- 拉镜像、启停容器

# Pod从敲命令到跑起来，完整流程是什么？
## 1.kubectl提交创建Pod请求→kube-apiserver
## 2.apiserver校验、认证，把Pod信息存入etcd
## 3.kube-scheduler发现有未调度的Pod计算合适节点，把节点名写入etcd
## 4.对应节点的kubelet观察到“有Pod分配给我了”
## 5.kubelet让容器运行时拉镜像、创建容器
## 6.kubelet汇报给apiserver
## 7.Pod运行成功
