# 一、Volume是什么？
- 让容器**数据持久化**，容器删了数据还在
- 解决：容器**重启/重建数据丢失**的问题
# 二、常用本地存储（面试常问）
## **1.emptyDir**
- 跟Pod同生命周期
- Pod删，数据删
- 用途：临时缓存、共享文件
### **2.hostPath**
- 把**节点本机目录**挂载进容器
- 数据存在节点上
- 风险：Pod飘到别的节点就找不到数据
- 一般只用于**日志、节点工具**
# 三、持久化存储核心三要素（必背）
## 1.**PV**（PersistentVolume）
- 集群里的**一块存储空间**
- 由管理员准备
## 2.**PVC**（PersistentVolumeClaim）
- 用户**申请存储**
- 像“点餐”：我要10G、读写模式
## 3.**StorageClass（SC）**
- **存储类**
- 自动创建 PV → **动态供给**
- 不用管理员手动创建PV
# 四、PV 三种回收策略（超级高频）
## **1.Retain（保留）**
- PVC删了，**PV还在，数据还在**
- 需要管理员手动清理
## **2.Delete（删除）**
- PVC删了，**PV也删，数据没了**
- 云存储常用
## 3.Recycle（回收）
- 旧版，基本淘汰
- 简单清数据后复用
# 五、静态供给VS动态供给
- **静态**：管理员提供建好一堆PV
- **动态**：通过**StorageClass**，用的时候自动创建 PV
# 六、访问模式（常问）
- ReadWriteOnce（RWO）
	- 只能被**一个节点**挂载读写
- ReadOnlyMany（ROX）
	- 多节点**读写**
- ReadWriteMany（RWX）
	- 多节点**同时读写**（NFS、Ceph等）
# 七、CSI是什么
- Container Storage Interface
- 存储插件标准
- 让各种存储（云盘、NFS、Ceph）接入K8s

# 面试高频
## 1. emptyDir 和 Pod 是什么关系？
和Pod同生命周期
## 2. PVC、PV、StorageClass 分别干嘛？
PV是集群的一块存储空间
## 3. PV 三种回收策略是什么？
Retain（保留）、Delete（删除）、Recycle（回收）
## 4. 动态供给靠什么实现？
通过StorageClass实现
