# 一、集群高可用（HA）
- 多Master节点，避免单点故障
- 核心组件高可用：
	- kube-apiserver多实例
	- etcd集群（3/5节点）
	- controller-manager、scheduler多副本竞选
- 前端加负载均衡（LB）统一入口
# 二、HPA水平自动扩缩容
- 依据CPU、内存或自定义指标
- 自动增加/减少Pod副本数
- 应对流量波动，保证性能
# 三、VPA垂直扩缩容
- 自动调整Pod的CPU、内存限制/请求
- 适合无法水平扩容的应用
# 四、节点亲和性/反亲和性
- **亲和性**：Pod尽量调度到同一个节点/机架
- **反亲和性**：Pod分散到不同节点
	- 避免节点全部挂掉导致服务不可用
# 五、污点（Taint）与忍耐（Toleration）
- 给节点打污点，拒绝普通Pod调度
- 特定Pod配置忍耐才能上去
- 用于：专用节点、GPU节点、隔离环境
# 六、资源限制（面试必写）
- requests：调度用，保证最低资源
- limits：上限，防止耗尽节点资源
- 必须配置，避免服务互相影响
# 七、Pod中断预算PDB
- 限制同时中断Pod数量
- 保证升级、维护时服务不雪崩
# 八、常见性能优化
- 使用IPVS代替iptables提升Service性能
- 合理设置requests/limits
- 避免节点资源利用率过高
- 日志、监控分离，不影响业务
- Etcd单独部署，SSD磁盘


# 面试高频题
1.K8s高可用核心是多什么？
- 多Master节点
- 多核心组件：
	- kube-apiserver、etcd集群、controller-manager、scheduler
2.HPA是干嘛的？
- 水平自动扩容
3.反亲和性作用是什么？
- 把Pod分散到不同节点，避免一个Pod挂了，导致整个服务挂
4.requests和limits区别？
- requests：是保证最低资源
- limits：是保证最高资源，避免耗尽资源
